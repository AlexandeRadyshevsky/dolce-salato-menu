# SEO: инструкция по проекту Dolce&Salato

Документ для SEO-специалиста. Описывает, где лежат настройки, что и как
правится, и как устроена генерация SEO при сборке.

## 1. Как устроен сайт (коротко)

Сайт собран на React + Vite. Это одностраничное приложение, но при сборке
для каждой категории и каждого товара создаётся отдельная HTML-страница с
человекопонятным адресом (ЧПУ), своим title, description, canonical и
микроразметкой. То есть для робота это полноценный набор страниц, а не один
пустой index.

Главная работает как лендинг с блоками: О нас, Доставка, Каталог,
Прайс-лист, Контакты. Каталог дальше разворачивается в отдельные страницы.

Структура адресов:

```
/                                              главная
/katalog/                                      список всех категорий
/katalog/pasta/                                страница категории
/katalog/pasta/spagetti-optom/                 страница товара
```

## 2. Главный файл, который ты правишь

Почти всё содержимое и SEO-поля лежат в одном файле:

```
src/data/menu.json
```

Меняешь значения в нём, делаешь git push, сайт пересобирается и сам
проставляет всё в HTML. Руками HTML-страницы трогать не нужно, они
генерируются.

## 3. Что и где ставить в menu.json

### Блок "site" (общие настройки и главная страница)

```json
"site": {
  "url": "https://dolcesalato.ru/",
  "title_ru": "...",
  "description_ru": "...",
  "keywords_ru": "...",
  "yandex_verification": "PASTE_YANDEX_WEBMASTER_CODE",
  "og_image": "images/og-cover.jpg"
}
```

- `url`: финальный адрес сайта с протоколом и слешем на конце. От него
  считаются все ссылки в sitemap, canonical и микроразметке. Это первое,
  что надо проставить правильно. Если адрес неверный, вся SEO-разметка
  будет ссылаться не туда.
- `title_ru`: заголовок главной (тег title). До 60 знаков.
- `description_ru`: описание главной (meta description). 140 - 160 знаков.
- `keywords_ru`: ключевые слова через запятую. Под Яндекс имеет смысл,
  держим актуальными.
- `yandex_verification`: код подтверждения прав из Яндекс.Вебмастера.
  Как получить: завести сайт в Вебмастере, выбрать способ "Мета-тег",
  скопировать значение content и вставить сюда. Сейчас стоит заглушка
  PASTE_YANDEX_WEBMASTER_CODE, её надо заменить.
- `og_image`: картинка для превью в соцсетях и мессенджерах, 1200 x 630.
  Файл лежит в public/images/og-cover.jpg, можно заменить на брендовый.

### Заголовки и описания категорий

Категории лежат в массиве `categories`. У каждой:

```json
{
  "id": "pasta",
  "name_ru": "Паста",
  "name_it": "Pasta",
  "slug": "pasta",
  "products": [ ... ]
}
```

- `name_ru`: название категории, используется в заголовке h1 и в title
  страницы категории.
- `slug`: кусок адреса для этой категории (латиница, без пробелов). Менять
  не рекомендую без необходимости: если поменяешь, старый адрес перестанет
  открываться. Если всё же меняешь, настрой редирект со старого на новый.

Title и description страницы категории собираются автоматически по шаблону
из name_ru, name_it и названий товаров. Если нужно управлять ими вручную,
скажи, добавлю отдельные поля title/description в категорию.

### Карточки товаров

Внутри `products` у каждого товара:

```json
{
  "id": "pasta-spaghetti",
  "name_ru": "Спагетти",
  "name_it": "Spaghetti",
  "slug": "spagetti-optom",
  "producer": "Pasta Gentile",
  "format": "500 г",
  "desc_ru": "Текст описания товара.",
  "pairing": "С чем сочетается"
}
```

- `name_ru`: название товара, идёт в h1 и title страницы товара.
- `slug`: адрес товара. По задаче во всех slug добавлено слово "optom".
  Правило то же, что у категорий: без необходимости не меняем.
- `producer`: производитель, выводится на карточке и в микроразметке как
  бренд.
- `format`: фасовка (например "500 г", "1 кг").
- `desc_ru`: описание. Это основной текст для SEO на странице товара. Чем
  подробнее и уникальнее, тем лучше. Сюда и стоит вписывать ключевые слова
  естественным языком.
- `pairing`: с чем сочетается, опционально.

Title и description страницы товара собираются автоматически из name_ru,
name_it, producer и desc_ru. Если нужно ручное управление, скажи, добавлю
поля.

### Контакты (идут в микроразметку Organization)

```json
"contact": {
  "address_ru": "141400, Химки, ул. Рабочая, д. 2, стр. 95 ...",
  "locality": "Химки",
  "region": "Московская область",
  "postal": "141400",
  "country": "RU",
  "phone": "+7 (495) 782 83 12",
  "email": "ds.zakaz@gmail.com"
}
```

Эти поля попадают в Schema.org разметку организации (адрес, телефон,
почта). Держим в актуальном виде.

## 4. Что генерируется автоматически при сборке

Когда проект собирается (npm run build, на GitHub это происходит при каждом
push), движок сам делает следующее:

1. Подставляет title, description, keywords, canonical и код Яндекса в
   index.html.
2. Добавляет микроразметку Schema.org в формате JSON-LD: Organization
   (компания, адрес, телефон), WebSite (с поиском), ItemList (список всех
   товаров со ссылками).
3. Создаёт по отдельной HTML-странице для каждой категории и каждого товара
   со своими title, description, canonical и разметкой Product.
4. Кладёт скрытый текстовый каталог в тег noscript, чтобы робот видел
   контент и ссылки даже без выполнения JavaScript.
5. Генерирует sitemap.xml со всеми адресами (главная, каталог, категории,
   товары).

Файлы, которые отвечают за эту логику (трогать не нужно, но чтобы понимать):

```
scripts/seo.mjs       мета-теги, JSON-LD, noscript-каталог
scripts/routes.mjs    генерация ЧПУ-страниц и sitemap
vite.config.js        запуск всего этого при сборке
```

## 5. robots.txt и sitemap

```
public/robots.txt     правила для роботов
public/sitemap.xml    карта сайта
```

В robots.txt нужно проверить строку Sitemap и адрес домена: там должен
стоять реальный домен сайта, а не заглушка. Для Яндекса в файле прописан
Clean-param, чтобы не плодились дубли по меткам и параметрам сортировки.

sitemap.xml пересобирается автоматически при сборке. Если нужно обновить
карту вручную, не пересобирая весь проект:

```
node scripts/gen-sitemap.mjs
```

## 6. Порядок первого запуска (чек-лист)

1. В menu.json в блоке site проставить реальный url.
2. Туда же вставить yandex_verification из Вебмастера.
3. В robots.txt поправить домен в строке Sitemap.
4. Проверить title и description главной, переписать под семантику.
5. Пройтись по desc_ru товаров, наполнить уникальными текстами.
6. При желании заменить public/images/og-cover.jpg на брендовую обложку.
7. git push, дождаться сборки.
8. В Яндекс.Вебмастере добавить сайт, подтвердить права, отправить
   sitemap.xml на индексацию.

## 7. Важные ограничения

- Сайт развёрнут на статическом хостинге (GitHub Pages). Серверного
  рендеринга нет. Для индексации этого достаточно: контент дублируется в
  noscript и в статических ЧПУ-страницах, плюс есть JSON-LD. Если в будущем
  понадобится полноценный SSR, нужен переезд на хостинг с Node (Vercel,
  Netlify, свой сервер).
- Цен на сайте нет, по товарам работает форма запроса. Это осознанное
  решение под B2B.
- При смене slug категории или товара старый адрес перестаёт работать.
  Если меняешь, нужен редирект 301 со старого на новый.

## 8. Что пока не сделано на стороне сайта (вне фронтенда)

Из общего ТЗ к фронтенду не относится и делается отдельно:

- Подключение CRM и рассылок (Unisender, DashaMail и т.п.). Форма уже
  собирает заявки вместе с выбранными категориями интересов, данные готовы
  под выгрузку и сегментацию.
- Настройка Яндекс.Директа и счётчика Метрики. Код Метрики, когда будет,
  вставляется в index.html (скажи, добавлю место под него).
